# Bash CLI generation reference

Pin every Bash CLI generated by this skill to the idioms below. Do not invent
alternative shapes — regenerate the same structure every run. This reference
targets Bash specifically (`#!/usr/bin/env bash`); Bash-only extensions
(arrays, `local`, `${var^^}`, `[[ ]]`) are allowed here — unlike the sibling
`posix-sh.md` reference, which must avoid them.

## 1. Argument parsing

Use the POSIX-mandated `getopts` builtin as the parsing engine, always with a
leading `:` in the optstring so `getopts` stays silent and errors are handled
explicitly in the `\?`/`:` arms:

```bash
while getopts ":hjno:" opt; do
  case "$opt" in
    h) usage; exit 0 ;;
    j) json_output=1 ;;
    n) no_input=1 ;;
    o) output="$OPTARG" ;;
    \?) echo "error: invalid option: -$OPTARG" >&2; usage >&2; exit 2 ;;
    :)  echo "error: option -$OPTARG requires an argument" >&2; usage >&2; exit 2 ;;
  esac
done
shift $((OPTIND - 1))
```

`getopts` has no native `--long-option` support. Handle `--help`/`--json`/
`--no-input`-style long flags with a manual pre-scan that translates each
recognized long option into its short-option equivalent, using a Bash array
(`args+=(...)`, `set -- "${args[@]}"`) before the `getopts` loop runs:

```bash
args=()
for arg in "$@"; do
  case "$arg" in
    --help)     args+=(-h) ;;
    --json)     args+=(-j) ;;
    --no-input) args+=(-n) ;;
    --output)   args+=(-o) ;;
    --output=*) args+=(-o "${arg#*=}") ;;
    --)         args+=(--); ;;
    --*)        echo "error: unknown option: $arg" >&2; usage >&2; exit 2 ;;
    *)          args+=("$arg") ;;
  esac
done
set -- "${args[@]}"
```

Do not reach for `getopt(1)` (external, GNU-only, non-portable across macOS
BSD userland) or a third-party arg-parsing library — the translate-then-
`getopts` pattern above is the pinned shape.

## 2. Project layout

Generate this exact tree:

```
myapp/
├── bin/
│   └── myapp
├── lib/
│   └── myapp.sh
├── completions/
│   └── myapp.bash
├── Formula/
│   └── myapp.rb
├── packaging/                 # optional, only if apt packaging is explicitly requested
│   └── deb/
│       ├── control
│       └── rules
├── test/
│   ├── myapp.bats
│   └── testdata/
│       └── help.golden
└── README.md
```

`lib/myapp.sh` is a sourced function file: every function defined in it must
have zero side effects at source-time — no top-level code execution, no
argument parsing, no I/O outside function bodies — so it can be `source`d in
isolation from bats tests or a REPL. `bin/myapp` is the thin executable
(`#!/usr/bin/env bash`, `set -euo pipefail`) that resolves its own directory,
sources `lib/myapp.sh`, parses arguments, and dispatches to the sourced
functions:

```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "$SCRIPT_DIR/../lib/myapp.sh"
```

Never inline business logic in `bin/myapp` beyond parsing, dispatch, and
output formatting — anything else belongs in `lib/myapp.sh` so it is testable
by sourcing alone.

## 3. Help text and completions

`getopts` does not generate help text. Hand-write a `usage()` function in
`lib/myapp.sh` (or `bin/myapp`, since it has no side effects until called),
print it on `-h`/`--help` with exit 0, and print it to stderr with exit 2 on
any argument error:

```bash
usage() {
  cat <<'EOF'
Usage: myapp [-h] [-j] [-n] [-o FILE] <command> [args...]

Options:
  -h, --help          Show this help message and exit
  -j, --json          Output machine-readable JSON
  -n, --no-input      Never prompt; fail if required input is missing
  -o, --output FILE   Write output to FILE instead of stdout
EOF
}
```

Ship a hand-rolled `completions/myapp.bash`, registered with `complete -F`:

```bash
# completions/myapp.bash
_myapp_completions() {
  local cur="${COMP_WORDS[COMP_CWORD]}"
  COMPREPLY=($(compgen -W "-h --help -j --json -n --no-input -o --output" -- "$cur"))
}
complete -F _myapp_completions myapp
```

Document sourcing it from `~/.bash_completion` or `/etc/bash_completion.d/`
(Homebrew installs it to `$(brew --prefix)/etc/bash_completion.d/` when the
formula declares `bash_completion.install "completions/myapp.bash"`).

## 4. NO_COLOR-aware output

Bash has zero automatic `NO_COLOR` support — hand-roll it every time. Resolve
color eligibility once near the top of `bin/myapp`, honoring both `NO_COLOR`
and a non-TTY destination before any ANSI code is ever emitted:

```bash
[[ -n "${NO_COLOR:-}" ]] && USE_COLOR=0 || USE_COLOR=1
[[ -t 1 ]] || USE_COLOR=0
```

Gate every ANSI emission on `USE_COLOR`, never emit escape codes unconditionally:

```bash
color() {
  local code="$1"; shift
  if [[ "$USE_COLOR" -eq 1 ]]; then
    printf '\033[%sm%s\033[0m' "$code" "$*"
  else
    printf '%s' "$*"
  fi
}
```

## 5. Exit codes

Frozen cross-paradigm contract — every generated CLI, regardless of language,
must honor this:

- `0` — success
- `1` — general/runtime error (failed operation, caught error condition)
- `2` — usage/argument error (bad flags, missing required argument)

`getopts` never exits on its own — map both failure arms explicitly, as
shown in section 1: `\?)` (invalid option) and `:)` (missing an option's
argument) both `exit 2` after printing `usage >&2`. Runtime failures inside
dispatched functions `exit 1`:

```bash
if ! do_work; then
  echo "error: do_work failed" >&2
  exit 1
fi
```

Success is the implicit `exit 0` at the end of `bin/myapp`, or an explicit
`exit 0` at the end of the script for clarity in longer scripts.

## 6. `--json` output and `--no-input`

`--json`: switch the render step only, keep exactly one call into
`lib/myapp.sh` regardless of output mode. Hand-build the JSON string with
`printf` — do not assume `jq` is a dependency unless the CLI already needs
`jq` for parsing structured input, in which case use `jq -n --arg` instead of
hand-built strings so both paths go through the same tool:

```bash
# no jq dependency assumed
printf '{"state":"%s"}\n' "$state"

# only if jq is already a stated dependency (e.g. for parsing input)
jq -n --arg state "$state" '{state: $state}'
```

`--no-input`: guard every interactive `read` call with the flag. If a
required value is missing and `--no-input` is set, fail at exit code `2`
(usage error — the invocation was incomplete for non-interactive use)
instead of hanging on a prompt:

```bash
if [[ -z "${name:-}" ]]; then
  if [[ "$no_input" -eq 1 ]]; then
    echo "error: --name is required with --no-input" >&2
    exit 2
  fi
  read -r -p "Project name: " name
fi
```

Never call `read` (interactive) without checking `no_input` first — an
unguarded prompt hangs CI and piped-stdin invocations.

## 7. stdout/stderr discipline

Route results — anything a downstream `|` pipe consumes — to stdout via
`echo`/`printf`. Route diagnostics, progress, and logs to stderr with the
explicit `>&2` redirect. Never mix the two in the same call:

```bash
echo "$result"                  # stdout: data
echo "fetching..." >&2          # stderr: progress/diagnostic
```

Keep this rule absolute even inside `lib/myapp.sh` functions: a function that
prints a log line to stdout by mistake corrupts `--json` output and breaks
piping into `jq`/`grep`/downstream tools.

## 8. Distribution

**Homebrew (primary).** Recommend a self-contained single-file build as the
simpler default: a build step that inlines `lib/myapp.sh` into `bin/myapp`
so the formula installs one file and the two-file `source` dance never has
to survive an install-path change:

```bash
# scripts/build.sh — produce a single-file dist/myapp with lib inlined
awk '
  /^source .*lib\/myapp\.sh"$/ { while ((getline line < "lib/myapp.sh") > 0) print line; next }
  { print }
' bin/myapp > dist/myapp
chmod +x dist/myapp
```

```ruby
class Myapp < Formula
  desc "..."
  homepage "https://example.com/myapp"
  url "https://example.com/myapp/archive/v0.1.0.tar.gz"
  sha256 "..."

  def install
    bin.install "dist/myapp" => "myapp"
    bash_completion.install "completions/myapp.bash"
  end

  test do
    system "#{bin}/myapp", "--help"
  end
end
```

If the two-file layout must ship as-is (e.g. `lib/myapp.sh` is reused by
other tools), install both and patch the `source` path with `inreplace`
instead:

```ruby
def install
  (pkgshare/"lib").install "lib/myapp.sh"
  inreplace "bin/myapp", '$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)', "\"#{pkgshare}\""
  bin.install "bin/myapp"
end
```

**apt (optional, document honestly as heavier-weight).** Most Bash CLIs skip
a native `.deb` — document Homebrew as primary and give apt-based systems a
manual fallback instead:

```bash
curl -fsSL https://example.com/myapp/releases/download/v0.1.0/myapp.tar.gz \
  | tar -xz -C /usr/local
```

Only generate `packaging/deb/control` + `packaging/deb/rules` when the user
explicitly asks for real apt packaging — a minimal `control` (Package,
Version, Architecture: all, Depends: bash, Description) plus a `dh $@`
`rules` file — and say plainly that this adds real ongoing maintenance
(changelog, versioning, repo hosting) that most single-binary Bash tools
don't need.

## 9. Snapshot testing `--help`

Use bats-core with a golden-file diff — never a hand-rolled string-equality
assertion. Generate `test/myapp.bats`:

```bash
@test "--help matches golden output" {
  run "$BATS_TEST_DIRNAME/../bin/myapp" --help
  [ "$status" -eq 0 ]
  diff <(echo "$output") "$BATS_TEST_DIRNAME/testdata/help.golden"
}
```

Generate the golden file from the real binary, never hand-typed:

```bash
bin/myapp --help > test/testdata/help.golden
```

Run with `bats test/` (install via `brew install bats-core`). Any unreviewed
`--help` drift then fails `diff` in CI instead of passing silently.

## 10. Minimal worked example

Illustrative only — the actual generation is always freeform from this
reference, never a copy-paste of this exact code.

```bash
# lib/myapp.sh — sourced function file, zero side effects at source time
greet() {
  local name="$1" shout="${2:-0}"
  if [[ -z "$name" ]]; then
    echo "error: name must not be empty" >&2
    return 1
  fi
  local text="Hello, ${name}!"
  [[ "$shout" -eq 1 ]] && text="${text^^}"
  printf '%s\n' "$text"
}
```

```bash
#!/usr/bin/env bash
# bin/myapp — thin executable: parse, dispatch, format
set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "$SCRIPT_DIR/../lib/myapp.sh"

usage() { printf 'Usage: myapp [-h] [-s] NAME\n'; }

args=(); for a in "$@"; do case "$a" in --help) args+=(-h) ;; --shout) args+=(-s) ;; *) args+=("$a") ;; esac; done
set -- "${args[@]}"

shout=0
while getopts ":hs" opt; do
  case "$opt" in
    h) usage; exit 0 ;;
    s) shout=1 ;;
    \?) echo "error: invalid option: -$OPTARG" >&2; usage >&2; exit 2 ;;
  esac
done
shift $((OPTIND - 1))
[[ $# -eq 1 ]] || { usage >&2; exit 2; }

greet "$1" "$shout" || exit 1
```
